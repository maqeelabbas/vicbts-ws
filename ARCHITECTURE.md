## ğŸ¯ Key Design Patterns

### 1. **APP_INITIALIZER Pattern**
Load configuration before app starts:
```typescript
{ provide: APP_INITIALIZER, useFactory: initFn, multi: true }
```

### 2. **Functional Guards**
Modern Angular route protection:
```typescript
export const authGuard: CanActivateFn = (route, state) => { ... }
```

### 3. **Standalone Components**
No NgModules, direct imports:
```typescript
@Component({ standalone: true, imports: [CommonModule] })
```

### 4. **Generic Services with Inheritance**
DRY principle for API services:
```typescript
export class UserService extends BaseApiService<User> { }
```

### 5. **Observable State Management**
Reactive data flow:
```typescript
config$ = new BehaviorSubject<AppConfig | null>(null);
```

### 6. **Barrel Exports (index.ts)**
Clean import paths:
```typescript
// libs/shared/ui/src/index.ts
export * from './lib/button/v-button.component';
```
